Welcome to Swift

About Swift

Swift 支持移动端, PC, 服务器, 或任何其他能够执行 Swift 的平台. 它安全, 快速, 吸收了 Apple 和开源社区的开发智慧. 对编译器的性能进行了优化, 同时语言对于开发更友好.

Swift 易于上手. 它是一门工业级编程语言, 与脚本语言类似的易于表达和令人享受. 使用支持 Swift 的 PlayGround , 可以在不导入任何头文件或者运行app的情况下测试代码, 并且立即看到结果.

Swift 吸取现代编程语言的经验, 通过定义大量的的类, 来避免普遍的编程错误.

  • 变量 在使用之前必须被实例化
  • 对 数组 索引进行越界检查
  • 对 整型 进行溢出检查
  • 使用 可选类型 来明确保证 nil 的持有情况
  • 自动内存管理
  • 错误处理保证了未知的错误变得可控

为了最大化的发挥硬件性能, Swift 的代码进行了优化. 它的语法和标准库基于”所见即所得”的原则被定义. 它的安全和快速使 Swift 成为编写 “Hello World!” 或者是一个完整应用的最佳选择.

Swift 将强大的类型推断和模式匹配融合到现代的, 轻量级的语法中, 让复杂的想法以简洁的方式表现出来. 得益于此, 代码变得易于读, 写和维护.

版本兼容性

本文基于 Swift 4.1, 默认包含于 Xcode 9.2. Xcode 9.2 可以编写 Swift4 或者 Swift3

注释
当 Swift 编译器在编译 Swift 3 的代码, 它会将语言版本认定为 3.2, 你可以使用条件语句 #if swift(>=3.2) 来能让多版本语言共存.

当你使用 Xcode 9.2 来编译 Swift 3 的代码, 大部分 Swift 4 特性都是可以使用的, 只有如下特性只支持 Swift 4:

  • Substring 操作返回值是 Substring 类型, 而不是 ‘String’类型.
  • @objc 属性在更少的地方被自动添加.
  • 在同一文件中对类型添加的 Extension 可以访问原类型的私有成员变量.

一个 Swift 4 Target 可以依赖于 Swift 3,反之同理. 也就意味着, 如果你以库的形式创建你的工程, 你可以一次性将代码从 Swift 3 迁移到 Swift 4.
